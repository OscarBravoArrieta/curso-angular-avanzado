Formateo de Código con Prettier y ESLint en Proyectos Angular
https://platzi.com/cursos/angular-avanzado/optimiza-tu-codigo-con-prettier-en-angular/

 1. Prettier is an automatic code formatter. Its main function is to rewrite your source code according to predetermined and consistent rules.
    npm install prettier -D
     a. update packaje.json
            ...
            "scripts": {
                "format": "prettier --write ."
            }

     the point (.) indicate all folder

     b. To format all project, execute: npm run format

     c. For ESLint and Prettier to work in harmony, it is necessary to install some additional packages:

        npm install prettier-eslint eslint-config-prettier eslint-plugin-prettier -D

        update: eslint.config.js

     d. Make a file .prettierrc.json
     {
        "tabWidth": 2,
        "useTabs": false,
        "singleQuote": true,
        "semi": true
     }

    ¿Cuál es la diferencia entre ESLint y Prettier?
    Antes de sumergirnos en la implementación, es importante entender la distinción entre estas dos herramientas fundamentales:
    ESLint: Se enfoca en analizar la calidad del código, identificando malas prácticas y problemas de accesibilidad. No está diseñado principalmente para formatear código.
    Prettier: Es un formateador de código que establece un estilo consistente en todo el proyecto. Su objetivo es que todos los desarrolladores escriban código con el mismo formato, independientemente de sus preferencias personales.

Configuración y uso del ambiente de staging en Angular
https://platzi.com/cursos/angular-avanzado/configura-entornos-staging-para-pruebas/

    1 Create environment.stagin.ts file in environments folder
    2. Update angular.json
        projects > store > architect > build > configuration >

    3. Ad the configuration
        "staging": {
            "fileReplacements": [
                {
                "replace": "src/environments/environment.ts",
                "with": "src/environments/environment.staging.ts"
                }
            ]
        },

    4. projects > store > architect > serve >
        "staging": {
            "buildTarget": "store:build:staging"
        },
    5. Execute stagin environment: ng serve -c stagin
